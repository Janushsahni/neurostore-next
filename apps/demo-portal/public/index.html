Enterprise Security • Browser Upload • Edge-Ready

Server never sees your plaintext files.

Files are encrypted in your browser before upload, then distributed across independent storage nodes. You keep the passphrase. Retrieval and decryption happen client-side.

Client-Side Encryption

AES-256-GCM with per-file random salt and nonce.

Decentralized Replication

Configurable replica factor across multiple node peers.

Failover Retrieval

Recover files even if a node goes offline.

How It Works

Simple workflow for teams and demos.

  1. Install node software via the Windows MSI on provider laptops.
  2. Collect each node's peer multiaddr and paste into upload form.
  3. Create account and login in this portal.
  4. Upload image/doc/pdf; browser encrypts before any network transfer.
  5. Retrieve + decrypt with passphrase; plaintext stays client-side.

Why Teams Use Neurostore

  • Zero-trust file handling in portal mode.
  • Node marketplace and decentralized redundancy.
  • Proof-based storage responses from nodes.
  • Built-in AI node risk and placement intelligence.
